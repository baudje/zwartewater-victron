CVL Active at Each Step — Crash Safety

T+0 — Normal Operation
Relay closed. CVL 28.4V from aggregate/99. LFPs on bus. Orion off.
T+1 — Register Temp Service
Temp service at 28.4V registered alongside aggregate. DVCC still reads agg/99 (lower instance wins). Safe for LFPs.
T+2-T+7 — Stop Aggregate Driver
DVCC now reads temp/100 at 28.4V. Relay still closed. Crash here: startup finds relay closed → idle.
T+7-T+17 — Open Relay, Verify
Relay opens. Orion activates. CVL still 28.4V. LFPs now on Orion. Crash here: relay open, delta likely <2V → auto-close.
T+17 — Raise CVL to EQ Voltage
Only after relay open verified. CVL raised to 31.2V. LFPs protected on Orion. Crash here: temp dies → no CVL → Quattro stops charging. LFPs still safe on Orion.
T+18-T+168 — Equalising FLA
Quattro charges Trojans at 31.2V. Monitor I_trojan. Detect Orion failure (V_lfp drop >0.5V). 2.5hr hard timeout.
T+169 — Cooling Down
EQ complete (I < 8A). CVL reduced to 27.6V (float). LFPs still on Orion. Quattro begins float mode.
T+199 — Voltage Matching
Wait for |V_trojan − V_lfp| < 1V. CVL 27.6V. Max 4hr timeout. SmartShunt Trojan must be responsive.
T+230 — Reconnect LFP
Close relay. Verify via read-back. Measure inrush current + reconnect delta. CVL 27.6V.
T+234 — Restart Aggregate Driver
Deregister temp service. Restart aggregate/99. Brief gap with no CVL — safe as relay is closed and both banks on bus.
T+244 — Normal Operation Restored
Aggregate/99 running. DVCC reads CVL 28.4V. Record timestamp. 90-day interval resets.
Key Safety Invariant

CVL is never raised above 28.4V while the relay is closed. The 31.2V equalisation voltage is only applied after relay open is verified with a D-Bus read-back. If the service crashes while equalising at 31.2V, the temp battery service disappears, the Quattro sees no CVL and stops charging — the LFPs remain safely powered by the Orion DC-DC.

Safety Guards & Recovery Actions

Guard Trigger Action
Crash-safe CVL Service crash before relay opens Temp service at 28.4V (safe for LFPs), never 31.2V while relay is closed
Delta-aware finally Any abort with relay open Only closes relay if |delta| < 2V; otherwise raises alarm and stays open
Startup recovery Service start with relay open Checks voltage delta, auto-closes (<2V) or alarms and stays open (≥2V)
Relay verification After relay open AND close commands Reads back D-Bus hardware state; aborts if read-back does not match command
Orion failure V_lfp drops > 0.5V during EQ Abort equalisation, reconnect LFP (delta-aware)
SmartShunt Trojan V_trojan = None (shunt offline) Abort in both Equalising and Voltage Matching states
Current loss i_trojan = None for 5 minutes Proceed to voltage matching (not full 2.5hr timeout)
High current warning Trojan I > 60A during EQ Warning only — dynamo or MPPT detected, continue EQ
Voltage match timeout > 4 hours in Voltage Matching Alarm raised, stay disconnected, manual intervention required
SoC enforcement RunNow with LFP SoC < 95% Refused — SoC requirement is never bypassed, even with RunNow
Inrush monitoring After relay reconnect Measures & logs inrush current + delta; displayed on web UI for calibration
Bounds validation Web UI setting change Checked against min/max bounds before D-Bus write
Double trigger RunNow race condition RunNow flag cleared when equalisation starts
LFP current check Before relay open Verify LFP current < 5A before opening relay to avoid large inrush on open

RunNow bypasses all conditions except SoC. The SoC requirement protects against equalising with depleted LFPs.

Error Recovery Sequence

1. Deregister temp battery service
2. Check relay state in finally block
3a. delta < 2V → auto-close relay
3b. delta ≥ 2V → alarm, stay open
4. Restart aggregate driver if stopped
LFPs remain safely on Orion DC-DC in either outcome until relay is manually closed.

Disconnecting LFP

State 2
Actions
1. Open Cerbo relay 2
2. Verify: LFP current < 5A
3. Read-back relay state (abort on mismatch)
4. Record V_lfp at disconnect
5. Raise CVL to 31.2V (only after verified open)
System state
Relay OPEN Orion ON CVL 28.4V → 31.2V
Critical invariant
CVL raised to 31.2V only after relay open is confirmed via read-back.

Equalising FLA

State 3
Actions
Monitor SmartShunt Trojan current. Detect Orion failure (V_lfp drop >0.5V). Warn if I_trojan >60A.
System state
Relay OPEN Orion ON CVL 31.2V
Exit conditions
I_trojan < 8A (EQ complete)
OR 2.5 hour hard timeout
OR i_trojan = None for 5 min
Crash safety
Temp service dies → no CVL → Quattro stops charging. LFPs still safe on Orion.

Cooling Down

State 4
Actions
Reduce CVL to 27.6V (float voltage). Trojans cool down at float. Wait for V_trojan to stabilise before voltage matching.
System state
Relay OPEN Orion ON CVL 27.6V
Transitions to
Voltage Matching (immediate, after CVL set)

Voltage Matching

State 5
Actions
Wait for |V_trojan − V_lfp| < 1V. Check SmartShunt Trojan is responsive. Poll voltage delta periodically.

Relay OPEN Orion ON CVL 27.6V
Exit conditions
delta < 1V → Reconnecting
Timeout >4hr → Alarm, stay open
V_trojan = None → Error
Max wait
4 hours. After that: alarm raised, relay stays open, manual intervention.

Reconnecting LFP

State 6
Actions
1. Close relay 2
2. Verify: relay state read-back = closed
3. Measure inrush current (SmartShunt LFP, 1s after close)
4. Record reconnect delta
System state
Relay CLOSED Orion OFF CVL 27.6V
Inrush calibration
Inrush <50A → delta threshold can increase. Inrush >100A → reduce delta threshold to protect relay contacts.
